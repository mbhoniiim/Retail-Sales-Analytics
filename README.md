Retail Sales Analytics Database

Project Overview

This project simulates a retail business database and demonstrates how SQL can be used to answer real-world business questions.

The database contains information about:

- Customers
- Products
- Orders
- Order Details

Using SQL queries, the project analyses sales performance, customer behaviour, product performance, and revenue trends.



Objectives

The goal of this project is to demonstrate:

- Database design
- Data modeling
- SQL querying skills
- Business analysis using SQL
- Preparation for Data Analyst and BI Analyst roles



Database Schema

Customers

Stores customer information.


2. Products

Stores product information.


3. Orders

Stores customer orders.


4. OrderDetails

Stores products purchased within each order.



Data Generation

Data was generated using:

- Mockaroo
- Custom SQL inserts

| Table | Rows |
|---------|---------|
| Customers | 500 |
| Products | 50 |
| Orders | 2,000 |
| OrderDetails | 10,000 |



Business Questions Answered

1. Top Customers by Spending

Identify customers generating the most revenue.

2. Top Selling Products

Determine the products with the highest sales volume.

3. Revenue by Category

Analyze which product categories generate the most revenue.

4. Monthly Revenue Trends

Track revenue performance over time.

5. Average Order Value

Measure the average value of customer purchases.

6. Best Performing Cities

Identify geographic areas generating the highest revenue.

7. Customer Lifetime Value

Calculate the total revenue generated by each customer.
